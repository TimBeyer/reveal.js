  1. 1993:
    HTML 1.0 specification released
  2. 1994:
    W3C is founded to promote open standards for the web
  3. 1995:
    Browser wars - Microsoft and Netscape battle for biggest browser market share
  4. April 1995:
    Brendan Eich creates JavaScript for Netscape in 10 days
  5. September 1995:
    Netscape releases JavaScript in the Netscape Navigator 2.0 beta
  6. August 1996:
    Microsoft releases a JavaScript clone called JScript for IE 3.0
  7. June 1997:
    W3C standardized JS as ECMA-262 (officially called ECMAScript now)
  8. December 1997:
    HTML 4 specification is published

So what happened?

How do browsers work?

  • Browser gets HTML from server
  • Browser parses HTML into a tree representation
  • Browser requests external files if mentioned in HTML (JS, stylesheets, images)
  • Layout Engine parses the HTML tree and stylesheets, renders image to screen
  • Browser runs JS
  • Browser waits for user interaction

The DOM

Some more history

  1. 1997:
    MS releases IE 4.0, Netscape releases Netscape 4.0 - Both with support for Dynamic HTML (DHTML)
  2. 1998:
    W3C publishes DOM Level 1 standard
  3. 2000:
    W3C publishes DOM Level 2 standard
  4. 2004:
    W3C publishes DOM Level 3 standard
- DHTML is the combination of JS, a rudimentary DOM and HTML - DHTML does not follow the same standard for all browsers - maintenance nightmare - DOM Level 1 allows for changing any portion of the document - Browsers still not all standards compliant, IE and Netscape 4.0 still widespread - DOM Level 2 introduces `getElementById`
